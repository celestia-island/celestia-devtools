# Соглашение о сообщениях коммитов

Все репозитории Celestia Island применяют соглашение о сообщениях коммитов на основе [gitmoji](https://gitmoji.dev). Этот документ определяет правила, исключения и механизмы принудительного применения.

## Правило

Тема каждого коммита (первая строка сообщения коммита) должна соответствовать следующему формату:

```
<gitmoji> <Описание на английском с заглавной буквы.>
```

| Требование | Пример (проходит) | Пример (не проходит) |
|---|---|---|
| Начинается с gitmoji | `🐛 Fix...` | `Fix...` |
| Без префикса Conventional Commits | `🐛 Fix...` | `🐛 fix: ...` |
| Первая буква после emoji заглавная | `🐛 Fix...` | `🐛 fix...` |
| Заканчивается точкой | `🐛 Fix the parser crash.` | `🐛 Fix the parser crash` |
| Только английский | `🐛 Fix the parser crash.` | `🐛 修复解析器崩溃。` |
| Описательно (не только версия/заполнитель) | `⬆️ Upgrade the HTTP client to v2.` | `⬆️ 0.3.0` |

## Исключения

Следующие типы коммитов автоматически освобождаются:

- **Слияния**: `Merge branch 'foo'` / `Merge pull request #42`
- **Откаты**: `Revert "..."`

Чтобы пропустить проверку для одного коммита:

```bash
CELESTIA_COMMIT_MSG_SKIP=1 git commit
```

## Принудительное применение

Соглашение применяется на трёх уровнях:

1. **Локальный хук commit-msg** — автоматически устанавливается командой `celestia-devtools init`. Блокирует недопустимые сообщения во время `git commit`. Для переустановки или обновления:

   ```bash
   celestia-devtools hook install
   celestia-devtools hook install --force   # перезаписать пользовательский хук
   just commit-msg-hook-install             # через just recipe
   ```

2. **Проверка CI (PR)** — повторно используемый рабочий процесс `commit-msg-lint.yml` проверяет каждый коммит в pull request. Добавьте эту задачу в `checks.yml` вашего репозитория:

   ```yaml
   commit-msg:
     uses: celestia-island/celestia-devtools/.github/workflows/commit-msg-lint.yml@master
   ```

3. **Защита ветки** — настройте проверку статуса `commit-msg` как обязательную для ветки `master` в настройках вашего репозитория GitHub.

## Репозитории ботов

Репозитории, полностью управляемые автоматизированными процессами (например, `provider-registry`), должны пропускать локальный хук, запуская `celestia-devtools init --no-hooks`, и пропускать проверку CI, не включая её. Отдельные коммиты ботов также могут быть исключены через `CELESTIA_COMMIT_MSG_SKIP=1`.

## Миграция

Существующие коммиты в истории репозитория не проверяются ретроактивно — это соглашение применяется только к новым коммитам. Если ветка master вашего репозитория в настоящее время использует префиксы Conventional Commits (`feat:`, `fix:`, и т.д.), вы можете постепенно перейти, приняв формат gitmoji для всей новой работы.
